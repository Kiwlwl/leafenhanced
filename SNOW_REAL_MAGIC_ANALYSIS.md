# Snow! Real Magic! 架构分析

> 分析目的：为本模组（落叶增强）寻找可借鉴的技术与设计方案。

## 1. 模组定位

- **类型**：小型原版增强模组。
- **核心机制**：增强原版雪的物理与视觉效果，不新增大量内容。
- **特点**：每个功能基本都可配置、可开关。

## 2. 核心机制

### 2.1 雪的方块套壳（Block Wrapping）

- 在原版方块（台阶、楼梯、栅栏等）上放置雪时，不替换原方块，而是“套上一层雪”。
- 需要保存原始方块状态，因此用到自定义方块实体或扩展方块状态。
- **对本模组的启示**：落叶增强不需要套壳机制。我们直接复用原版 `leaf_litter` 方块在地上生成即可。

### 2.2 层状雪

- 雪可以像蛋糕一样堆叠层数，最多 8 层。
- 下雪时层数自动增加。
- **对本模组的启示**：落叶层也可以用类似层数逻辑，优先在已有 `leaf_litter` 上继续堆积。

### 2.3 暴风雪（Blizzard）

- **不是新的天气类型**。
- 通过游戏规则（gamerule）增强下雪效果：粒子更密、下落速度更快、视野受限。
- 本质是原版雪的强化表现。
- **对本模组的启示**：不需要新增天气/季节。落叶效果可以直接与群系绑定，保持简单。

### 2.4 落雪方块实体

- 雪会以“下落方块实体”形式从空中落下并堆积。
- **对本模组的启示**：落叶粒子只做客户端视觉效果，不需要真实下落方块实体。

## 3. 技术实现

- **Mixin**：大量修改原版方块与实体行为。
- **配置**：可配置几乎所有效果（粒子密度、暴风雪强度、雪融化等）。
- **客户端/服务端分离**：视觉粒子在客户端，方块状态变化由服务端驱动。

## 4. 可借鉴的设计

| Snow Real Magic | 落叶增强对应设计 |
|---|---|
| 增强原版雪 | 增强原版树叶与落叶层 |
| 可配置一切 | 每个功能都有配置项 |
| 方块套壳保存原方块 | 直接复用原版 `leaf_litter`，不套壳 |
| 层状雪堆积 | 落叶层层状堆积 |
| 暴雪 = 强化原版雪 | 落叶 = 与群系绑定，全年发生 |
| 客户端粒子 + 服务端方块 | 客户端落叶粒子 + 服务端随机 tick 生成落叶层 |

## 5. 不应照搬的部分

- **方块套壳**：过于复杂，落叶不需要保存原方块。
- **新增天气/游戏规则**：原版没有四季，强行加季节系统会增大复杂度。
- **下落方块实体**：落叶粒子用客户端粒子即可，不需要物理下落实体。

## 6. 结论

Snow! Real Magic! 最值得借鉴的是：

1. **小而精**的定位。
2. **可配置、可开关**的设计哲学。
3. **客户端粒子 + 服务端方块状态**的分离实现。
4. **层状堆积**的可视化思路。

我们的落叶模组可以走更简单的路线：复用原版方块、按群系触发、不做方块套壳、不做季节系统。
