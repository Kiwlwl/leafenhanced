# 风力系统改进计划

> 当前 MVP 已完成基础风向，以下为未来可改进方向。

## 1. 阵风（Gust）

### 现象
风不是恒定不变的，而是偶尔突然增强再减弱——一阵风吹过，叶子猛地横向飘散。

### 实现
```
正常 windTarget: ±baseWindStrength
阵风触发: random 每 200~600 tick 检查一次
阵风期间: windTarget 临时增大到 baseWindStrength × 3~5
          持续 20~60 tick 后恢复
```

### 效果
叶子在阵风时集体横向漂移，风停后逐渐恢复自然飘落。

---

## 2. 湍流 / 风噪声（Turbulence）

### 现象
风的方向不是完全平滑的，而是带有微小的抖动——叶子飘落时轻微颤抖。

### 实现
```
每 tick: windX += 小幅度随机噪声 × 0.05
```

### 效果
叶子轨迹不再是平滑弧线，而是略带颤抖，更接近真实叶片下落。

---

## 3. 高度风切变（Wind Shear）

### 现象
树冠处的风比地面大，高处叶子飘得更远，地面附近几乎无风。

### 实现
```
风对粒子的影响 = baseWind × (粒子Y - 地面Y) / 最大高度
或简化为: 低于玩家 Y 的风力减半
```

### 效果
树顶叶子被吹得很远，快落到地面时几乎垂直下落。

---

## 4. 局部风遮蔽（Wind Shading）

### 现象
建筑物、山体背风面的叶子不受风影响。

### 实现
```
从粒子位置向 windX/windZ 方向做方块追踪
如果碰到固体方块，说明在背风面，减少风力影响
```

### 效果
房子一侧落叶纷飞，另一侧几乎没有风——增加沉浸感。

---

## 5. 风向地图（Flow Field）

### 现象
开阔地带和树林中的风向不同，风绕过树木流动。

### 实现
```
基于地形预计算一个低分辨率的风向场
粒子根据当前位置查表获得风力
```
计算量大，性能开销较高。

### 效果
最真实，但复杂度高，暂不建议。

---

## 优先级建议

1. 阵风（最明显的效果提升，最简单）
2. 湍流（几行代码，效果明显）
3. 高度风切变（实现简单，提升真实感）
4. 局部风遮蔽（中等难度，边界情况多）
5. 风向地图（不推荐，除非有性能余量）
